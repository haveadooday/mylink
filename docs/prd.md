# 마이링크 (MyLink) 서비스 기획서 (PRD)

## 1. 프로젝트 개요
- **프로젝트명**: 마이링크 (MyLink)
- **목적**: 여러 개의 링크(SNS채널, 포트폴리오, 동영상, 블로그 등)를 하나의 통합된 웹페이지로 모아서 공유할 수 있는 멀티 링크 프로필 서비스 제공.
- **대상 사용자**:
  - 다양한 채널(인스타그램, 유튜브 등)을 운영하며 하나로 통합해 홍보하고 싶은 크리에이터 및 인플루언서
  - 포트폴리오 및 깃허브 링크들을 한 곳에 깔끔하게 정리하고자 하는 취업 준비생(개발자, 디자이너 등)
  - 제품/서비스 홍보 채널을 한곳에 모으려는 소상공인

---

## 2. 핵심 기능 목록 및 요구사항

### 2.1. 인증 및 계정 관리
- **[필수] 소셜 로그인 (Google)**
  - Firebase Authentication을 활용한 빠르고 안전한 구글 단일 소셜 로그인.
  - 별도의 복잡한 이메일/비밀번호 가입 절차 없이 구글 계정으로 즉시 가입 및 로그인 처리.
- **[필수] 로그아웃 / 계정 탈퇴**
  - 사용자 정보 안전 처리를 위한 세션 종료 기능 및 데이터 영구 삭제 기능.

### 2.2. 사용자 프로필 설정
- **[필수] 프로필 정보 작성 및 수정**
  - 사용자에게 노출될 **닉네임** 및 **한 줄 소개(Bio)** 텍스트 작성 (가입 후에도 언제든지 자유롭게 마이페이지에서 수정 가능).
  - *(참고: 별도의 프로필 이미지 업로드 기능은 제공하지 않음)*
- **[필수] 고유 링크(URL) 설정**
  - 별도의 핸들(handle) 부여 없이, 사용자가 설정한 **닉네임을 그대로 기반으로** 고유한 마이링크 주소를 생성 (e.g., `마이링크.com/닉네임`).

### 2.3. 링크 관리 (에디터)
- **[필수] 기본 링크 추가/수정/삭제**
  - 사용자가 표시할 '타이틀'과 이동할 'URL'을 입력하여 버튼 형태로 된 링크 블록을 생성 및 관리.
- **[필수] 확장 링크 유형 지원**
  - 단순 URL 이동 외에도 **다양한 형태의 임베드** 지원 (예: 화면 내에서 바로 재생 가능한 YouTube 영상 임베드).
  - 링크 외에 단순 안내나 텍스트를 적을 수 있는 **텍스트 블록** 지원.
- **[필수] 링크 아이템 파비콘(Favicon) 연동**
  - **Google Favicon API**를 활용하여, 사용자가 등록한 URL의 파비콘 이미지를 자동으로 가져와 해당 링크 버튼의 아이콘으로 표시.
- **[필수] 링크 활성화/비활성화 제어 (토글)**
  - 등록된 각 링크 항목을 삭제하지 않고도 방문자 화면에서 일시적으로 노출 또는 숨김 처리할 수 있는 On/Off 기능 제공.
- **[필수] 드래그 앤 드롭 정렬**
  - 관리자(마이페이지) 화면에서 마우스를 이용해 링크 블록들의 위아래 노출 순서를 직관적으로 변경.

### 2.4. 소셜 아이콘 퀵 링크 
- **[필수] 플랫폼 전용 아이콘형 링크**
  - 텍스트 버튼으로 넣기엔 공간을 많이 차지하는 주요 SNS 및 연락처(Email, YouTube, Instagram, GitHub 등)를 프로필 상/하단에 아이콘(뱃지) 형태로 작게 배치하는 기능.

### 2.5. 프로필 공유 기능
- **[필수] 링크 간편 복사**
  - 원클릭으로 자신의 마이링크 주소를 클립보드에 복사하여 다른 플랫폼에 즉시 붙여넣기 가능.

## 3. 추후 추가 예정 기능 (Phase 2)
### 3.1. 대시보드 및 통계
- **조회수 및 클릭수 추적**
  - 내 프로필 페이지의 **총 방문자 수(조회수)** 지표 제공.
  - 사용자가 등록한 각 개별 링크 및 아이콘 항목의 **클릭 횟수** 통계 제공.

---

## 4. 요약 및 개발 기준
- **플랫폼 환경**: 모바일 화면 중심의 **반응형 웹 애플리케이션 (Responsive Web)**.
- **기반 기술(Tech Stack) 검토**:
  - **Front-end**: React / Next.js (사용자 친화적 디자인과 SSR/SEO 고려시 유리)
  - **Back-end & DB**: Firebase (Authentication을 통한 구글 로그인 처리 및 Firestore 데이터베이스 연동)
